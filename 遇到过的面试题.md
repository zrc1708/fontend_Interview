# 单页面应用的优缺点
+ 优点
    + 良好的交互体验，内容的改变不需要重新加载整个页面，使用更流畅
    + 良好的前后端分离模式，后端不用负责模板渲染，后端api通用化
+ 缺点
    + 首屏加载慢，加载首页就会将所有组件全部加载（解决方法：路由懒加载，cdn加速，异步加载组件。。。）
    + 不利于SEO，搜索引擎请求到的html是模型页面而不是最终数据的渲染页面。 这样就很不利于内容被搜索引擎搜索到。
    + 不适合大型项目
